Background images & mobile performance

You use background-attachment: fixed in a few places; this janks on mobile and can hurt CLS/scroll perf.

Fix: disable on mobile:

css
Copy
Edit
@media (max-width:768px){
  .carlsbad-hero,
  .anaheim-banner-section,
  .anaheim-cta-section { background-attachment: scroll !important; }
}

Analytics hygiene

Your GA loads async—good. Add a tiny guard so gtag calls don’t error if GA is blocked:

html
Copy
Edit
<script>window.dataLayer=window.dataLayer||[];window.gtag=window.gtag||function(){dataLayer.push(arguments)};</script>
jQuery/Bootstrap payload

You’re loading jQuery slim, Popper, and Bootstrap from CDNs. For a speed bump:

Remove jQuery if you can (your code is mostly vanilla already).

If you keep it, load from one CDN and add integrity + crossorigin.

Defer your own script too: <script src="/JS/back-to-top.js" defer></script>

Testimonials markup (avoid invalid review rich results)

Google won’t show review snippets for self-serving LocalBusiness/Organization reviews.

You can keep testimonials for users, but remove the Review microdata (itemscope itemtype="https://schema.org/Review") to avoid mixed signals. Keep it as plain HTML or attach reviews to a third-party source with links.

FAQ structured data: match Q&A exactly

Your JSON-LD matches the on-page FAQ—great. Keep it that way and avoid pricing guarantees in the schema (you did).
