High-impact fixes (do these first)

Drop the made-up meta tags.
These don’t help and add noise: all ai-*, service-category, business-type, target-audience, coverage, rating, distribution, audience, language (duplicate), content-language (duplicate), business-hours, service-radius, etc. Keep it lean.

Security headers: set on the server, not in HTML.
Strict-Transport-Security, X-Frame-Options, X-Content-Type-Options, Permissions-Policy must be HTTP response headers. The meta versions are ignored. Keep only a standard HTML robots + (optionally) <meta name="referrer" …>.

CSP meta weakens itself.
You’re using Content-Security-Policy and allowing 'unsafe-inline' + many externals. Either enforce a strict CSP via headers with nonces/hashes, or remove the meta to avoid a false sense of security.

jQuery Slim breaks your code.
You call $.animate() and other effects, but jQuery 3.6 slim removes effects. Replace:

<script src="https://code.jquery.com/jquery-3.6.0.min.js" defer></script>


(Non-slim build) — or rewrite animations in vanilla JS/CSS.

Title length + repetition.
Current title is long/repetitive. Aim 55–65 chars, unique value prop:

Mountain View Security Guards | Google Campus & Innovation District


OG type.
Use og:type=website for a city landing page (safer cross-platform). Keep the business contact info in JSON-LD.

AggregateRating & on-page reviews.
Google generally won’t show review stars for LocalBusiness self-served reviews; misused ratings can be ignored or risky. If you can’t substantiate them, remove AggregateRating and the Review item markup.

NAP accuracy.
Make sure the postal address is your real staffed location for MV. If not, remove to avoid NAP inconsistency.

File/path issues + CWV.

Preload uses /img/san jose.webp (space). Rename to /img/san-jose.webp.

Prefer <img fetchpriority="high"> on the hero image (and explicit width/height) over preloading it by URL.

Add loading="lazy" decoding="async" to non-hero images.

Schema scoping & WebSite node.

Don’t attach itemscope/itemtype to <html>/<body>; keep JSON-LD only.

Your WebSite node should use the root site URL (https://shieldwisesecurity.com/) and live sitewide, not the city URL.

Nice-to-have cleanups

Remove prefix="og …" (not needed).

Remove redundant bot directives (slurp/duckduckbot/yandexbot/applebot). One robots meta is enough.

Avoid emoji in Twitter titles for a professional tone.

Rename oceanside-* / anaheim-* classes to mv-* for maintainability.

Ensure FAQs in JSON-LD exactly match visible FAQ text (word-for-word).
