Fix these first (highest impact)
Two H1s
You have the hero <h1> and another <h1> (“Professional Security Guard Services…”). Keep one H1; make the second an <h2>.

Obsolete/ignored meta tags
Remove: keywords, both revisit-after (duplicate too), rating, expires, generator, distribution, geo.*, ICBM, DC.title, duplicate language. They add noise.

Security headers in HTML
HSTS, X-Frame-Options, Permissions-Policy, etc. do nothing in <meta http-equiv>. Set them on the server response, not in markup.

Hero LCP is a CSS background
Browsers can’t prioritize it well. Use a real <img>/<picture> in the hero with fetchpriority="high" + width/height to avoid CLS. Keep your gradient overlay with a pseudo-element.

</picture>
JSON-LD correctness

aggregateRating + review: only include if those reviews are rendered on-page and are yours. Avoid listing Google/Yelp as publisher.

"elevation" is not valid for GeoCoordinates → remove.

priceSpecification.price must be a number, not "35-65". Use separate Offers or a minimum price.

Add separate Organization and WebSite graphs (keep your LocalBusiness/SecurityService node).

Preload / fonts mismatch
You preload Inter, but body uses a system stack. Either add Inter to font-family or drop the preload. Also, font-display: swap belongs in @font-face, not on body (you can ignore if using Google Fonts’ display=swap param—which you have ✅).

Performance & UX tweaks
Background parallax / background-attachment: fixed hurts paint, especially on mobile. You already turn it off ≤768px—consider dropping it entirely or using a light transform-based effect.

jQuery + Bootstrap + Font Awesome are heavy. If possible, replace with tiny, self-hosted CSS and SVG icons. If keeping them, self-host and preload critical CSS.

Add width/height and loading="lazy" to all non-hero images to prevent CLS.

modulepreload only if /js/critical.js and /js/lazy-loading.js are ES modules. Otherwise use:

IntersectionObserver selector bug: you observe .anaheim-service-item, but your cards use .anaheim-service-item-enhanced. Update the selector so animations actually fire.
