What’s hurting (and how to fix fast)

Title is too long (likely truncates).
Aim ~55–60 chars.
Fix (recommended):

<title>Sacramento Security Guards | Armed & Unarmed 24/7</title>


Alt longer brand variant:

<title>Sacramento Security Guards | 24/7 Armed & Unarmed | ShieldWise</title>


Meta keywords + legacy geo tags = noise.
meta name="keywords", geo.*, ICBM, locality, region, country-name don’t help and add bloat.
Remove those tags entirely.

Security headers via <meta http-equiv> don’t work.
X-Frame-Options, X-XSS-Protection, Permissions-Policy, Referrer-Policy must be sent by the server, not HTML.
Action: move to server config (Nginx/Apache/Cloudflare). Keep HTML clean.

Open Graph type
business.business is poorly supported.
Fix:

<meta property="og:type" content="website">


(Keep a separate LocalBusiness JSON-LD for local signals—which you already have.)

Hreflang set is incomplete.
You have en-US and es-US, but no x-default. Also ensure the Spanish page points back with reciprocal tags.
Fix:

<link rel="alternate" hreflang="en-US" href="https://shieldwisesecurity.com/security-guards-sacramento">
<link rel="alternate" hreflang="es-US" href="https://shieldwisesecurity.com/es/guardias-seguridad-sacramento">
<link rel="alternate" hreflang="x-default" href="https://shieldwisesecurity.com/security-guards-sacramento">


Multiple <h1>s.
You have an H1 in the hero and another “Top Security Guard Services in Sacramento”. Use one H1.
Fix: keep the hero as <h1>, change the inner one to <h2>.

Your LocalBusiness uses @id pointing to /#organization, but your Organization doesn’t expose the same @id. Link them.

areaServed entries show "@type": "City" with string containedInPlace. Use Places or just strings.

aggregateRating requires review evidence on page. If you don’t render ~198 reviews/ratings, remove or reduce claims.

Claims need proof (E-E-A-T).
Statements like “zero incidents,” “98% patient satisfaction,” “198 reviews,” “largest team,” can trigger quality checks.
Action: either (a) render evidence (logos with permissions, case-study pages, testimonial schema), or (b) soften language (“including…”, “such as…”, “rated highly by clients”).
